# Product

## Register

product

## Users

PCShop serves first-time and entry-level PC buyers who understand budgets and use cases better than socket names, wattage margins, or case clearances. They are usually choosing parts on a phone, comparing a few options, and trying to avoid a wrong purchase without needing to become hardware experts.

## Product Purpose

PCShop is a mobile-first PC build quotation tool. It helps users select eight core part categories, blocks hard compatibility conflicts, surfaces soft warnings, and produces a quote that can be saved, shared, copied, or printed. Success means a novice user can move from budget intent to a complete, purchasable configuration with visible confidence in compatibility.

## Brand Personality

Calm, precise, protective. The interface should feel like a knowledgeable shop assistant who quietly prevents mistakes, explains only what matters, and keeps the buying flow moving.

## Anti-references

Avoid dense spreadsheet-like parts tables, hover-dependent desktop configurators, cart-first shopping flows that hide compatibility until checkout, and raw backend tag labels leaking into customer-facing UI. The product should not look like a parts database wearing a thin storefront skin.

## Design Principles

- Make progress visible at all times so users never need to remember what is missing.
- Block hard conflicts immediately and explain the exact reason in plain language.
- Reveal part details progressively: name, price, and key specs first; complete specifications only on demand.
- Let structured fields and backend-resolved actions drive behavior, while keeping internal tag keys out of the customer interface.
- Favor thumb-friendly mobile flows first, with desktop gaining density rather than different behavior.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Support keyboard navigation, visible focus states, screen-reader labels, reduced-motion preferences, and clear state text for selected, unavailable, incompatible, and warning states. Copy should assume no hardware background and avoid abbreviations unless the product data already uses them as specs.
